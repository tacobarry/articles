Introdução

O strict mode é uma feature disponível desde o ECMAScript 5. Através dela é possível optar por uma variante restrita do JavaScript, onde seu código será interpretado de modo mais rigoroso. Neste modo, a engine do JavaScript tem seu comportamento modificado, gerando erros que antes eram silenciados e, até mesmo, proibindo o uso de certas partes da linguagem que são tidas como problemáticas ou imaturas (sintaxes de versões futuras do ECMAScript), nos forçando assim a escrever um código de melhor qualidade e ajudando a capturar bugs mais precocemente.

Existem navegadores que não suportam o strict mode em sua plenitude (conforme a página Compat Table), o que muitas vezes proporciona comportamentos lamentáveis. Portanto, caso sua aplicação possua uma gama peculiar de browsers, fique atento a suas respectivas compatibilidades. Código strict mode e código não-strict mode podem coexistir, então scripts podem optar pelo strict mode incrementalmente.

Invocando strict mode

Scrict mode pode ser usado em scripts ou funções individuais. Elementos como eval, function, atributos de event handlers, strings passadas para setTimeout e coisas do tipo são scripts e, por isso, o strict mode funcionará adequadamente.

Strict mode para scripts

Para invocar o strict mode em um script inteiro, simplesmente coloque a diretiva "use strict"; (ou 'use strict';) antes de qualquer outra declaração: 

// Sintaxe strict mode para todo o script
"use strict";
var v = "Oi! Eu sou um script strict mode!";

Mas cuidado! Essa abordagem pode ser uma cilada, visto que ao concatenar scripts podem ocorrer conflitos indesejados. A Amazon que o diga, a empresa passou por problemas relacionados a isso, vejam dois "bugs" que deram o que falar: Bug 579119 e Bug 627531).

Por isso, lembre-se:

  • Strict Mode + Não-Strict Mode = Strict Mode
  • Não-Strict Mode + Strict Mode = Não-Strict Mode

Alternativamente, a concatenação de tipos com a mesma definição (strict ou não-sctrict) acarretará em um script final equivalente. Portanto é recomendado que você ative strict mode função-por-função (pelo menos durante o período de transição).

Strict mode para funções

Da mesma forma, para invocar strict mode em uma função, coloque a diretiva "use strict"; (ou 'use strict';) no corpo da função antes de qualquer outra declaração. 

function strict(){
  // Sintaxe strict mode a nível de função
  'use strict';
  function nested() { return "E eu também!"; }
  return "Oi! Eu sou uma função strict mode!  " + nested();
}
function notStrict() { return "Eu não sou strict."; }

Por que usar?

Torna mais fácil e natural o desenvolvimento de JavaScript "seguro".

Apresenta erro em "sintaxes inadequadas" (más práticas), geralmente aceitas em situções normais.

Em JavaScript não-strict, você não será avisado caso crie uma variável global por engano (veja essa discussão). Já no strict mode, isto irá lançar um erro, tornando impossível criar acidentalmente uma variável global.

Enfim, o strict mode é importante para que sejamos guiados às tendências e boas práticas de mercado. Além disso, o feedback imediato da diretiva previne equivocos graves em nossos scripts.
