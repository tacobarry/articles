Introdução

JavaScript é uma linguagem fracamente tipada onde os tipos de dados são convertidos dinamicamente durante a execução dos scripts.

O padrão ECMAScript atual define 7 tipos de dados, sendo 6 primitivos e Object.

Neste artigo entenderemos um pouco mais sobre os tipos de dados e suas principais características.

Tipos Primitivos

Dentre os 6 tipos primitivos temos:

  1. Boolean
  2. null
  3. undefined
  4. Number
  5. String
  6. Synbol

É interessente observar que sendo uma linguagem fracamente tipada com seus tipos de dados convertidos automaticamente durante a execução do script, é possível que uma mesma variável referencie dois ou mais tipos distintos de dados durante sua existência.

Exemplo: 

    let value;

    value = 'Eu sou um texto';
    typeof value;
    // $ "string"

    value = 1337;
    typeof value;
    // $ "number"

Outro ponto interessante é que o contexto influencia diretamente na conversão dos valores.

Exemplo: 

'22' + 2;
// $ '222'

'22' - 2;
// $ 20

Quanto empregamos o operador de soma sobre operandos do tipo String e Number. O operando do tipo Number é convertido para String. Já para os demais operadores aritméticos como subtração, multiplicação ou divisão a String é convertida para Number.

Vale lembrar que para cenários em que precisamos converter de String para Number podemos utilizar dois métodos parseInt(String, base) onde o parâmetro base é opicional e indica o sistema numérico a ser utilizado e parseFloat(String).

Boolean

O tipo boolean é o tipo mais simples de dados e o mesmo possui dois valores literais (true/false) além do wrapper Boolean.

Vale resaltar que em JavaScript as seguintes expressões são evaluadas como false:

  • false
  • 0
  • ''
  • ""
  • NaN
  • null
  • undefined

Exemplo: 

if(!("Não é um número" - 22)) { // equivalente a !NaN
    console.log('entrou!');
}

// $ entrou!

null

Em JavaScript null é o tipo primitivo que indica a ausencia de valor.

undefined

Em JavaScript undefined representa a inexistência de uma determinada propriedade.

Number

Em JavaScript todos os números são agrupados pelo tipo Number. Além de representar os números o mesmo possui três valores simbólicos, +Infinity, -Infinity e NaN. Isso mesmo, por mais estranho que pareça, typeof NaN é Number!

Um ponto de atenção é que o JavaScript, bem como outras linguagens, utiliza o padrão Double Precision 64-bit binary format IEEE 754 para representação dos números.

Este padrão pode gerar alguns resíduos, como no exemplo abaixo.

Exemplo: 

0.1 + 0.2;
// $ 0.30000000000000004

O "00000000000000004" é um residuo decorrente da operação binária. Veja mais sobre esta operação binária aqui.

Parece estranho, mas fiquem calmos! Para compreender e utilizar as principais funções de manipulação de números em JavaScript consulte este material e para compreender a Math Api colsulte este material .

Synbol

Definido na ECMAScript 6, representa um tipo de dado cuja as instancias são imutáveis e únicas. Para mais informações consulte este material .

String

Strings são compostas por um conjunto de caracteres e representam um texto, são objetos imutaveis e sua declaração se dá a partir de aspas duplas ou aspas simples.

Eu particularmente utilizo, sempre que possível, aspas simples para a declaração de Strings pois quando estamos trabalhando com HTML utilizamos aspas duplas e esta prática ajuda a evitar conflitos.

Para saber mais sobre os métodos que o Wrapper de String fornece recomendo este material da w3schools .

Object

Em JavaScript os objetos são estruturas de dados dinâmicas chave valor que comportam qualquer tipo de dados.

Neste artigo veremos alguns conceitos básicos e futuramente, em outros artigos da série iremos nos aprofundar mais, principalmente em orientação a protótipo.

Para construir um objeto podemos utilizar a forma literal.

Exemplo: 

    let obj = {};

A qualquer momento podemos adicionar e remover propriedades de um determinado Objeto.

Exemplo: 

    let pessoa = {
        nome: 'Renan'
    };

    // Cria propriedade dinamicamente
    pessoa.idade = 24;

    // Remove propriedade nome
    delete pessoa.nome;

Propriedades de um determinado objeto podem ser outros objetos ou funções.

Exemplo: 

    let pessoa = {
        nome: 'Renan',
        endereco: {
          logradouro: 'José Inocencio de Camargo Lima Filho',
          numero: '223'
        },
        falar: function() {
            console.log(this.nome);
        }
    };

    pessoa.falar();

    // $ Renan

Pelo fato dos objetos em JavaScript serem uma coleção de dados, podemos acessar seus valores através da notação de array.

Este tipo de notação se mostra muito útil quando precisamos acessar dinâmicamente as propriedades dos objetos.

Exemplo: 

    var pessoa = {
        nome: 'Renan',
    }

    pessoa['idade'] = 24;

    for (var prop in pessoa) {
        console.log( pessoa[prop] );
    }

    // $ Renan
    // $ 24
